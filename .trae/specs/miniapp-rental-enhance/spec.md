# 小程序租赁平台增强功能 - Product Requirement Document

## Overview
- **Summary**: 对现有小程序房屋租赁管理平台进行功能增强，包含省市县行政区划管理、字典标签体系、有效期与次数管理、消息推送、审批详情页优化、电话簿分类树、轮播图管理、字典管理优化、参数配置优化等11项核心功能。
- **Purpose**: 提升平台的地区管理能力、用户体验和运营效率，为后续业务扩展提供基础支撑。
- **Target Users**: 平台管理员、运营人员、小程序用户

## Goals
- 建立完整的省市县行政区划数据管理体系，支持小区登记开关控制
- 扩展字典体系，支持用户、房屋、小区、商家多维度标签
- 建立统一的有效期与次数管理机制，支持商城兑换延长和后台手动调整
- 支持管理员主动向小程序用户推送消息
- 优化审批页面交互，增加折叠详情和时间轴展示
- 优化电话簿管理，增加商家分类树导航
- 新增轮播图广告管理功能
- 优化字典管理和参数配置界面，提升运营效率

## Non-Goals (Out of Scope)
- 不涉及小程序端 UI 页面开发（仅管理后台）
- 不重构现有业务模块的核心逻辑
- 不涉及支付、订单等交易相关功能
- 不开发数据分析与报表功能

## Background & Context
- 基于若依(RuoYi) Vue 管理框架，前后端分离架构
- 已有的业务模块：小区管理、房屋管理、小程序用户、电话簿、合同、积分商城等
- 现有技术栈：Spring Boot + MyBatis + Vue + Element UI
- 数据库：MySQL

## Functional Requirements

### FR-1: 省市县基础数据管理
- 新建 sys_region 表存储全国省市县三级行政区划数据
- 字段：region_id, parent_id, region_code, region_name, region_level, community_register_switch, sort, status
- 支持数据导入（SQL 初始化脚本）
- 管理后台支持增删改查、按层级查询
- 小区登记开关：可按省/市/县级别开启或关闭小区登记功能

### FR-2: 小区登记地区校验与默认值
- 小区登记（含申请）时省市县默认选择：山东省 / 聊城市 / 阳谷县
- 提交时校验所选地区的小区登记开关是否开启
- 若开关关闭，提示"该地区未开放使用，请联系平台"
- 小区表 province/city/district 字段存储编码，展示时转换为名称

### FR-3: 字典标签体系扩展
- 在 sys_dict_type 中新增四类字典：
  - biz_user_tag：用户标签
  - biz_house_tag：房屋标签（南北通透、干净整洁、空调、wifi、密码锁、热水器、油烟机、可携带宠物等）
  - biz_community_tag：小区标签（绿化面积大、物业服务好、小区清洁等）
  - biz_merchant_tag：商家标签
- 房屋、小区、小程序用户实体增加 tags 字段存储标签编码（逗号分隔）
- 管理后台列表和详情页支持标签展示

### FR-4: 有效期与次数管理
- 房屋表增加 valid_until 字段（有效期截止日期）
- 电话簿表增加 valid_until 字段
- 小程序用户表增加 publish_count（发布次数）、publish_period_end（周期结束时间）字段
- 用户默认登记时消耗发布次数
- 商城虚拟商品（如"发布次数延长卡"）兑换后可延长有效期或增加次数，支持叠加
- 后台可手动调整用户发布次数和有效期

### FR-5: 小程序用户消息推送
- 新增 biz_user_message 表（消息记录表）
- 管理后台小程序用户列表增加"推送消息"按钮
- 支持推送标题和内容
- 记录推送时间、推送人、已读状态
- 提供消息列表查询接口（供小程序端调用）

### FR-6: 审批页面优化（折叠详情 + 时间轴）
- 小区登记审批页、房屋登记审批页左侧增加折叠按钮
- 点击展开后显示详细信息
- 详情区域增加横向时间轴，展示审批流程节点（提交→待审批→通过/驳回）
- 时间轴节点显示时间和操作人

### FR-7: 小程序用户详情折叠
- 小程序用户管理列表增加折叠展开按钮
- 展开后显示用户详细信息（手机号、认证状态、真实姓名、身份证号、登录信息等）
- 折叠状态下仅显示基本信息（头像、昵称、用户类型、状态）

### FR-8: 电话簿商家分类树
- 电话簿管理页面左侧增加商家分类树
- 分类树支持展开/折叠
- 点击分类节点筛选右侧电话簿列表
- 分类数据来源于字典（biz_merchant_category）或独立表

### FR-9: 轮播图管理
- 新增 biz_banner 表
- 字段：banner_id, image, title, content, valid_start, valid_end, contact_phone, remark, sort, status
- 管理后台支持增删改查
- 支持图片上传
- 支持有效期设置（发布有效期）
- 提供小程序端查询有效轮播图的接口

### FR-10: 字典管理界面优化
- 字典类型列表左侧增加折叠按钮
- 点击展开后显示该字典的明细数据
- 明细默认显示 10 条
- 可手动选择显示数量（10/20/50/100）
- 明细支持分页（上一页、下一页）
- 明细区域支持快速添加字典数据

### FR-11: 参数设置键选项配置
- sys_config 表增加 config_options 字段（参数键选项配置）
- 格式：参数名&参数值#参数名&参数值#...（如 "蓝色&skin-blue#绿色&skin-green#紫色&skin-purple"）
- 新建/编辑参数时，可配置参数键选项
- 配置后，参数键值以下拉选择形式展示（显示参数名，存储参数值）
- 对现有参数数据进行清洗，解析备注中的选项信息填充到 config_options 字段
  - 主框架页-默认皮肤样式名称 → 蓝色&skin-blue#绿色&skin-green#紫色&skin-purple#红色&skin-red#黄色&skin-yellow
  - 其他有枚举值的参数同理处理

## Non-Functional Requirements
- **NFR-1**: 行政区划数据导入后，三级联动查询响应时间 < 200ms
- **NFR-2**: 所有新增页面遵循现有若依框架的代码规范和 UI 风格
- **NFR-3**: 数据库表设计遵循现有命名规范（biz_ 前缀业务表，sys_ 前缀系统表）
- **NFR-4**: 新增接口遵循现有 RESTful 风格和权限控制机制

## Constraints
- **Technical**: 基于若依框架现有技术栈（Spring Boot + MyBatis + Vue 2 + Element UI）
- **Business**: 不影响现有业务模块的正常运行
- **Dependencies**: 依赖现有字典管理、参数配置、文件上传等基础功能

## Assumptions
- 全国省市县数据使用标准的中华人民共和国行政区划代码（GB/T 2260）
- 房屋登记审批页面已存在，需在现有基础上优化
- 商城兑换虚拟商品延长功能与现有积分兑换流程集成
- 消息推送仅为站内消息，不涉及微信模板消息/订阅消息

## Acceptance Criteria

### AC-1: 省市县数据管理
- **Given**: 管理员登录后台
- **When**: 进入行政区划管理页面
- **Then**: 可以查看省市区三级树状列表，可以新增/编辑/删除，可以设置小区登记开关
- **Verification**: `programmatic`

### AC-2: 小区登记地区校验
- **Given**: 用户在小区登记页面
- **When**: 选择省市县并提交
- **Then**: 系统校验该地区小区登记开关，关闭则提示"该地区未开放使用，请联系平台"
- **Verification**: `programmatic`

### AC-3: 默认地区选择
- **Given**: 用户打开小区登记页面
- **When**: 页面加载完成
- **Then**: 省市县默认选中 山东省 / 聊城市 / 阳谷县
- **Verification**: `programmatic`

### AC-4: 字典标签体系
- **Given**: 管理员在字典管理中配置了各类标签
- **When**: 在房屋/小区/用户编辑页面
- **Then**: 可以选择对应类型的标签，保存后列表和详情页正确展示标签
- **Verification**: `programmatic`

### AC-5: 有效期与次数管理
- **Given**: 房屋/电话簿有有效期设置
- **When**: 超过有效期后
- **Then**: 相关内容自动下架或标记过期
- **Verification**: `programmatic`

### AC-6: 商城兑换延长次数
- **Given**: 用户兑换了"发布次数延长"虚拟商品
- **When**: 兑换成功后
- **Then**: 用户发布次数相应增加，支持多次兑换叠加
- **Verification**: `programmatic`

### AC-7: 手动调整次数和有效期
- **Given**: 管理员在用户管理页面
- **When**: 点击"调整次数"或"延长有效期"
- **Then**: 可以手动修改用户的发布次数和有效期
- **Verification**: `programmatic`

### AC-8: 用户消息推送
- **Given**: 管理员在小程序用户列表
- **When**: 选择用户并推送消息
- **Then**: 用户收到消息，消息记录可查询
- **Verification**: `programmatic`

### AC-9: 审批页面折叠详情与时间轴
- **Given**: 管理员在审批列表页面
- **When**: 点击左侧折叠按钮
- **Then**: 展开显示详情和横向时间轴，时间轴展示审批流程各节点
- **Verification**: `human-judgment`

### AC-10: 用户详情折叠
- **Given**: 管理员在小程序用户列表
- **When**: 点击折叠按钮
- **Then**: 展开显示用户详细信息，折叠后仅显示基本信息
- **Verification**: `human-judgment`

### AC-11: 电话簿分类树
- **Given**: 管理员在电话簿管理页面
- **When**: 左侧点击某个分类
- **Then**: 右侧列表筛选显示该分类下的电话簿
- **Verification**: `programmatic`

### AC-12: 轮播图管理
- **Given**: 管理员在轮播图管理页面
- **When**: 新增/编辑轮播图并设置有效期
- **Then**: 有效期内的轮播图可在小程序端查询到，过期的不显示
- **Verification**: `programmatic`

### AC-13: 字典管理明细折叠
- **Given**: 管理员在字典类型列表
- **When**: 点击折叠按钮展开明细
- **Then**: 显示该字典的明细数据，默认10条，可调整每页数量，可翻页，可快速添加
- **Verification**: `human-judgment`

### AC-14: 参数键选项配置
- **Given**: 管理员编辑参数配置
- **When**: 配置了参数键选项
- **Then**: 参数值输入框变为下拉选择，显示选项名，保存选项值
- **Verification**: `programmatic`

### AC-15: 现有参数数据清洗
- **Given**: 系统中已有的参数配置
- **When**: 执行数据清洗脚本后
- **Then**: 有枚举值的参数的 config_options 字段被正确填充
- **Verification**: `programmatic`
