# 租赁平台标签/区划/角色/维修优化 - Product Requirement Document

## Overview
- **Summary**: 对租赁管理平台的标签选择、标签样式、角色管理、消息推送、行政区划展示、房屋维修等 6 个模块进行 Bug 修复与体验优化，提升管理后台的可用性和数据展示效率。
- **Purpose**: 修复已知交互 Bug（标签选择无反应、维修按钮无反应），优化列表页标签视觉样式，为角色管理增加行展开用户列表，为消息推送增加已推送列表与已读状态，为行政区划管理增加懒加载以缓解大数据量性能压力，将维修进度时间轴改为横向布局。
- **Target Users**: 平台运营管理员、后台运维人员

## Goals
- 修复小区/房屋/用户标签多选选择无响应的问题，确保标签可正常选择并保存
- 重新设计小区/房屋/用户列表标签展示样式，提升视觉效果与可读性
- 角色管理列表增加行展开功能，展开后显示该角色下的用户列表
- 小程序用户推送消息功能增加已推送消息列表，支持查看消息内容与是否已读状态
- 行政区划管理改为懒加载树形展示，降低一次性加载 3000+ 数据的性能压力
- 修复房屋维修"租客自修"、"房东确认维修"按钮点击无反应的问题，维修进度时间轴改为横向

## Non-Goals (Out of Scope)
- 不涉及小程序端 UI 改造（仅管理后台）
- 不涉及数据库表结构变更（仅使用现有字段）
- 不涉及用户权限体系重构
- 不涉及标签字典数据的新增或修改
- 不实现消息推送的真实微信推送能力（仅管理后台记录与展示）

## Background & Context
- 当前项目基于 RuoYi Vue Admin 框架（Spring Boot + MyBatis + Vue 2 + Element UI）
- 小区、房屋、用户三个模块均使用字典标签（biz_community_tag、biz_house_tag、biz_user_tag），前端使用 el-select multiple 多选
- 行政区划 sys_region 表当前有 3417 条记录（34 省 / 383 市 / 3000 区县），管理页一次性全量加载性能较差
- 后端已有 listByParentId 接口支持按父级查询子区划，但前端未启用懒加载
- 房屋维修前端调用的 API 路径与后端 Controller 实际路径不匹配，导致按钮点击后请求 404
- 角色管理已有 authUser/allocatedList 等接口用于查询角色关联用户

## Functional Requirements
- **FR-1**: 修复小区/房屋/用户标签选择无反应 Bug
  - 排查并修复 el-select multiple 在对话框中的选择无响应问题
  - 确保选择的标签能正确保存到数据库（逗号分隔字符串）
  - 确保编辑回显时已选标签正确显示
- **FR-2**: 小区/房屋/用户列表标签样式美化
  - 重新设计标签展示组件样式，增加圆角、渐变背景、图标等视觉元素
  - 优化标签间距、悬停效果、颜色搭配
  - 保持与现有 DictTag 组件的兼容
- **FR-3**: 角色管理行展开显示用户列表
  - 角色列表 el-table 增加 type="expand" 列
  - 展开后调用 authUser/allocatedList 接口加载该角色下的用户列表
  - 展示用户头像、昵称、手机号、用户类型等信息
  - 支持分页或限制展示数量，避免角色用户过多时展开区过高
- **FR-4**: 小程序用户推送消息 - 已推送列表与已读状态
  - 小程序用户管理页面增加"已推送消息"入口（行展开或独立弹窗）
  - 调用 listMessage 接口查询该用户已接收的消息列表
  - 列表展示消息标题、内容、发送时间、是否已读（0/1）、阅读时间
  - 消息项支持区分已读/未读样式
- **FR-5**: 行政区划管理懒加载优化
  - 行政区划 el-table 树形展示改为懒加载模式（lazy + load）
  - 初始仅加载省级数据，点击展开时按需加载下级区划
  - 保留搜索功能（搜索时全量匹配）
  - 配合后端 listByParentId 接口实现
- **FR-6**: 房屋维修按钮修复与时间轴横向
  - 修复"房东确认维修"、"租客自修"等按钮 API 路径不匹配的问题
  - 确保所有维修状态变更按钮均能正确调用后端接口
  - 维修进度时间轴改为横向展示（el-timeline 横向布局）
  - 保持时间节点的状态颜色与已完成/待处理状态对应

## Non-Functional Requirements
- **NFR-1**: 行政区划懒加载后，首屏加载时间减少 50% 以上
- **NFR-2**: 标签样式更新后保持与整体 Element UI 设计风格一致
- **NFR-3**: 角色行展开用户列表加载时间 < 2 秒
- **NFR-4**: 所有修复功能在 Chrome / Firefox / Edge 浏览器下正常工作

## Constraints
- **Technical**: Vue 2 + Element UI 2.x，不升级框架版本
- **Business**: 不修改现有业务流程，仅修复 Bug 与优化展示
- **Dependencies**: 依赖后端已有接口（listByParentId、authUser/allocatedList、listMessage 等）

## Assumptions
- 标签选择无反应的原因是 el-select 在 el-dialog 中 append-to-body 或 popper-class 相关问题
- 后端 listByParentId 接口可直接用于前端懒加载
- 角色用户列表使用已有的 authUser/allocatedList 接口
- 消息推送列表使用已有的 listMessage 接口，按 userId 筛选

## Acceptance Criteria

### AC-1: 标签选择功能正常
- **Given**: 用户在小区/房屋/用户编辑对话框中
- **When**: 点击标签多选下拉框并选择/取消标签
- **Then**: 下拉框正常展开，选中项正确显示，选择结果能正常保存并回显
- **Verification**: `programmatic`

### AC-2: 标签样式美化
- **Given**: 用户查看小区/房屋/用户列表的标签列
- **When**: 页面加载完成
- **Then**: 标签具有更美观的圆角、渐变/纯色背景、合适间距，视觉效果更好
- **Verification**: `human-judgment`

### AC-3: 角色管理行展开用户列表
- **Given**: 用户在角色管理列表页
- **When**: 点击某行的展开箭头
- **Then**: 展开区域显示该角色下的用户列表（头像、昵称、手机号等），数据正确加载
- **Verification**: `programmatic`

### AC-4: 消息推送已推送列表
- **Given**: 用户在小程序用户管理页面，对某用户点击查看已推送消息
- **When**: 展开或打开已推送消息列表
- **Then**: 显示该用户已接收的所有消息，包含标题、内容、发送时间、已读状态
- **Verification**: `programmatic`

### AC-5: 行政区划懒加载
- **Given**: 用户打开行政区划管理页面
- **When**: 页面初始加载
- **Then**: 仅加载省级数据（34 条），点击展开时才加载子级，首屏加载速度明显提升
- **Verification**: `programmatic`

### AC-6: 房屋维修按钮可用与横向时间轴
- **Given**: 用户在房屋维修列表页
- **When**: 点击"房东确认维修"或"租客自修"按钮
- **Then**: 请求成功，状态正确变更，列表刷新
- **Verification**: `programmatic`

### AC-7: 维修进度横向时间轴
- **Given**: 用户展开某条维修记录的详情
- **When**: 查看维修进度时间轴
- **Then**: 时间轴横向展示，节点从左到右按时间顺序排列，已完成节点绿色高亮
- **Verification**: `human-judgment`
