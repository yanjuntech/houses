# 租赁平台合同/地图/标签/小区/邀请/自动化测试优化 - Product Requirement Document

## Overview
- **Summary**: 对租赁管理平台的电子合同支付周期、小区地图显示、用户标签显示、房屋小区选择、邀请人管理展示、以及全业务流程自动化测试共 6 个方面进行功能增强与 Bug 修复。
- **Purpose**: 修复地图不显示、标签不显示等已知问题，将合同支付周期改为字典驱动的下拉选择，提升房屋小区选择的输入匹配体验，优化邀请人管理页面展示设计，并通过浏览器自动化测试确保业务管理下所有菜单和按钮流程可用。
- **Target Users**: 平台运营管理员、后台运维人员

## Goals
- 电子合同支付周期改为字典驱动的下拉选择，便于统一管理
- 修复小区地图不能正确显示的问题，增加加载失败友好提示
- 修复小程序用户维护标签后列表不能正确显示的问题
- 房屋管理小区选择支持输入过滤匹配
- 优化邀请人管理页面展示设计，状态不再显示为数字
- 通过浏览器自动化测试验证业务管理下所有菜单、按钮、流程可用，无 400 错误

## Non-Goals (Out of Scope)
- 不申请或配置真实的高德/腾讯地图 API Key（仅提供配置方式和降级方案）
- 不涉及后端业务逻辑重构
- 不修改数据库表结构（仅新增字典数据）
- 不实现移动端 UI 改造
- 自动化测试不涉及性能测试和压力测试
- 自动化测试不测试非业务管理模块（如系统管理、系统监控等）

## Background & Context
- 当前项目基于 RuoYi Vue Admin 框架（Spring Boot + MyBatis + Vue 2 + Element UI）
- 电子合同支付周期（payCycle）当前为自由文本输入，缺乏统一规范
- 小区地图选点组件 AMapPicker/TMapPicker 使用占位符 API Key，实际环境地图无法加载
- 小程序用户标签存储为逗号分隔字符串，需确认与字典值的对应关系
- 房屋管理小区选择为纯下拉选择，不支持输入搜索
- 邀请人管理页面较简单，需优化展示效果
- 业务管理模块包含多个子菜单：小区管理、房屋管理、电子合同、房屋维修、电话簿、轮播图、邀请管理、消息管理等
- 已有 dogfood/QA 测试技能和 agent-browser 工具可用于自动化测试

## Functional Requirements
- **FR-1**: 电子合同支付周期字典化
  - 新增字典类型 `biz_contract_pay_cycle`（支付周期）
  - 字典项包含：押一付一、押一付二、押一付三、押一付六、半年付、年付、季付
  - 合同编辑表单支付周期改为 `el-select` 下拉选择
  - 合同列表支付周期列使用 `dict-tag` 展示
  - 合同详情页支付周期使用 `dict-tag` 展示
- **FR-2**: 小区地图显示优化
  - 地图 API Key 改为从系统参数 `sys.community.mapKeyAmap` / `sys.community.mapKeyTencent` 读取
  - 在 sys_config 表中新增这两个配置项
  - AMapPicker/TMapPicker 组件从配置中读取 Key，不再硬编码
  - 地图加载失败时显示友好提示（如"地图加载失败，请检查 API Key 配置"）
  - 地图加载失败时仍可手动输入经纬度和地址
  - 增加地图加载状态（loading 动画）
- **FR-3**: 小程序用户标签显示修复
  - 排查并修复用户列表标签列不能正确显示的问题
  - 确保标签存储格式与字典值一致
  - 编辑回显时标签正确选中
  - 列表展示时标签正确渲染为 dict-tag
- **FR-4**: 房屋管理小区选择支持输入匹配
  - 房屋查询条件的小区下拉框增加 `filterable` 属性
  - 房屋编辑表单的小区下拉框增加 `filterable` 属性
  - 支持输入小区名称模糊匹配过滤选项
- **FR-5**: 邀请人管理展示优化
  - 修复邀请状态显示为数字的问题（确保 dict-tag 正确渲染）
  - 优化列表展示：增加行展开显示详细信息
  - 优化顶部统计卡片样式（美化总邀请数、已认证数）
  - 增加认证率展示（百分比）
  - 优化整体页面视觉效果（间距、对齐、配色）
- **FR-6**: 业务管理全流程自动化测试
  - 使用 agent-browser 内置浏览器进行模拟点击测试
  - 测试范围：业务管理下所有菜单项
  - 测试内容：每个页面的加载、所有按钮点击、主要流程操作
  - 验证标准：不出现 400/404/500 等错误提示
  - 生成测试报告，记录每个页面的测试结果
  - 测试页面包含：小区管理、房屋管理、电子合同、房屋维修、电话簿管理、轮播图管理、邀请管理、消息管理、小区登记申请、电话簿申请、小程序用户等

## Non-Functional Requirements
- **NFR-1**: 自动化测试覆盖业务管理至少 10 个页面
- **NFR-2**: 每个页面至少测试 3 个交互点（按钮、搜索、展开等）
- **NFR-3**: 地图组件加载失败降级方案不影响表单其他功能
- **NFR-4**: 所有页面在 Chrome 浏览器下正常工作

## Constraints
- **Technical**: Vue 2 + Element UI 2.x，不升级框架
- **Business**: 不修改现有业务流程，仅修复 Bug 与优化展示
- **Dependencies**: 依赖 sys_config 系统参数表、sys_dict 字典表
- **测试依赖**: 依赖后端服务和前端 dev server 正常运行

## Assumptions
- 支付周期字典值使用数字字符串（'0','1','2'...），与现有字典风格一致
- 地图 API Key 通过系统参数配置，用户自行申请填入
- 用户标签显示问题是由于存储值与字典值不匹配导致
- 房屋小区选择数据量不大时 filterable 性能可接受
- 自动化测试时数据库中有足够的测试数据

## Acceptance Criteria

### AC-1: 合同支付周期下拉选择
- **Given**: 用户在电子合同编辑对话框中
- **When**: 点击支付周期字段
- **Then**: 显示下拉选项，包含所有预设的支付周期类型
- **Verification**: `programmatic`

### AC-2: 合同列表支付周期展示
- **Given**: 用户查看电子合同列表
- **When**: 页面加载完成
- **Then**: 支付周期列显示为标签样式（dict-tag），而非纯文本
- **Verification**: `programmatic`

### AC-3: 地图加载失败友好提示
- **Given**: 地图 API Key 未配置或无效
- **When**: 用户打开小区编辑对话框的地图选点区域
- **Then**: 显示友好的错误提示信息，告知用户需要配置 API Key
- **Verification**: `programmatic`

### AC-4: 地图 API Key 可配置
- **Given**: 系统管理员
- **When**: 在系统参数管理中设置地图 API Key
- **Then**: 小区编辑页面地图组件使用配置的 Key 加载地图
- **Verification**: `programmatic`

### AC-5: 用户标签正确显示
- **Given**: 用户有标签数据
- **When**: 查看小程序用户列表的标签列
- **Then**: 标签正确渲染为 dict-tag 样式，显示对应的标签名称
- **Verification**: `programmatic`

### AC-6: 小区选择支持输入过滤
- **Given**: 用户在房屋管理页面
- **When**: 在小区下拉框中输入文字
- **Then**: 下拉选项根据输入内容自动过滤匹配的小区
- **Verification**: `programmatic`

### AC-7: 邀请状态正确显示
- **Given**: 邀请人管理列表有数据
- **When**: 页面加载完成
- **Then**: 邀请状态列显示为标签样式（已注册/已认证），而非数字
- **Verification**: `programmatic`

### AC-8: 邀请管理页面优化
- **Given**: 用户查看邀请人管理页面
- **When**: 页面加载完成
- **Then**: 页面有统计卡片、行展开详情、认证率等优化内容
- **Verification**: `human-judgment`

### AC-9: 自动化测试覆盖
- **Given**: 后端和前端服务正常运行
- **When**: 运行业务管理全流程自动化测试
- **Then**: 测试覆盖至少 10 个业务管理页面，每个页面至少 3 个交互点
- **Verification**: `programmatic`

### AC-10: 自动化测试无 400 错误
- **Given**: 自动化测试运行中
- **When**: 点击各种按钮和执行各种操作
- **Then**: 不出现 400/404/500 等错误提示弹窗
- **Verification**: `programmatic`
