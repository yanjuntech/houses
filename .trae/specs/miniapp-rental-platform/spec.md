# 微信小程序房屋租赁平台后台 Spec

## Why
为微信小程序提供房屋租赁（含租赁/求租/合租/出租/出售）业务后台支撑。基于已部署的若依（RuoYi-Vue 3.9.2 / Spring Boot 4.0.6 / MyBatis / Spring Security / JWT）框架扩展业务模块，避免重复造轮子，快速实现小区管理、房屋管理、电话簿管理、小程序用户管理及电子合同签署等能力。

## What Changes
- 新增**小程序用户管理**模块：手机号登录、微信快捷登录（登录后必须绑定手机号）、用户身份标签（房东/中介）、实名认证状态
- 新增**房屋租赁**菜单及业务模块：小区管理（含用户申请登记 + 审批流程）
- 新增**电话簿管理**模块：商家电话收录（含营业执照、店主信息登记 + 审批流程）
- 新增**房屋管理**模块：房源分类（在租/已租/上架/下架/出售/求租/合租）、按小区/户型筛选、上下架管理、房东与中介标签区分
- 新增**电子合同**模块：在线电子签名签署、合同 PDF 生成与下载
- 新增数据库表结构 SQL 脚本，集成至若依 `sql/` 目录
- 新增若依菜单配置（SQL 脚本方式注入 `sys_menu`），通过菜单权限控制访问
- 前端（`ruoyi-ui`）新增对应业务页面：列表查询、审批操作、详情查看、合同预览下载

## Impact
- Affected specs: 无（首次创建业务 spec）
- Affected code:
  - 后端：`ruoyi-system` 模块新增 `domain/mapper/service/impl`，`ruoyi-admin` 模块新增 `controller`
  - 前端：`ruoyi-ui/src/api/` 新增业务接口文件，`ruoyi-ui/src/views/` 新增业务页面
  - 数据库：新增 `sql/miniapp_rental.sql` 脚本
  - 配置：文件上传路径使用若依已有 `ruoyi.profile` 配置（`/home/ruoyi/uploadPath`）
  - 权限：通过若依菜单权限系统（`sys_menu` + `sys_role_menu`）控制
  - 认证：小程序端使用独立 JWT token（复用若依 `token.header` 配置），后端通过自定义 `@PreAuthorize` 注解控制接口权限

## ADDED Requirements

### Requirement: 小程序用户管理
系统 SHALL 提供小程序用户管理功能，支持手机号 + 验证码登录和微信快捷登录两种方式。微信登录后必须绑定手机号才能完成注册/登录。用户可被标记为"房东"或"中介"身份（通过标签区分），并记录实名认证状态。

#### Scenario: 手机号登录
- **WHEN** 用户输入手机号并获取验证码后提交登录
- **THEN** 系统校验验证码，成功后返回 JWT token，已注册用户直接登录，未注册用户自动注册并返回 token

#### Scenario: 微信快捷登录绑定手机号
- **WHEN** 用户选择微信登录获取 openid 后
- **THEN** 系统检查该 openid 是否已绑定手机号，未绑定时返回"需要绑定手机号"状态码，前端跳转绑定手机号页面
- **AND WHEN** 用户输入手机号 + 验证码完成绑定
- **THEN** 系统将 openid 与手机号关联，返回 JWT token

#### Scenario: 用户身份标签管理
- **WHEN** 管理员在后台为用户设置身份标签（房东/中介）
- **THEN** 用户记录更新 `user_type` 字段，小程序端发布房源时展示对应标签（房东直租/房屋中介）

#### Scenario: 实名认证状态管理
- **WHEN** 用户在小程序提交实名信息（姓名 + 身份证号）或管理员在后台审核
- **THEN** 系统更新 `verify_status` 字段（0未认证/1待审核/2已认证/3已拒绝）

### Requirement: 小区管理（含申请登记审批）
系统 SHALL 提供小区基础数据管理，并支持小程序用户申请新增小区。申请需经后台管理员审批通过后才能在系统中展示和使用。

#### Scenario: 用户申请登记小区
- **WHEN** 小程序用户搜索小区未果时提交小区申请登记（小区名、地址、所在区域）
- **THEN** 系统创建申请记录，状态为"待审批"，通知管理员
- **AND WHEN** 管理员在后台审批通过
- **THEN** 系统将申请数据写入 `biz_community` 表，状态置为"已审批"，申请人可在小程序中使用该小区

#### Scenario: 管理员驳回申请
- **WHEN** 管理员审批驳回并填写驳回原因
- **THEN** 申请记录状态置为"已驳回"，小程序端通知申请人

#### Scenario: 小区基础维护
- **WHEN** 管理员在后台维护小区信息（名称、地址、经纬度、物业公司）
- **THEN** 小区信息更新，关联房源同步显示最新小区名

### Requirement: 房屋管理
系统 SHALL 提供房屋（房源）信息管理，支持按租赁类型（出租/求租/合租/出售）、状态（在租/已租/上架/下架）、小区、户型（几室几厅）等多维度筛选查询，并支持上下架管理和房屋信息维护。每条房源 MUST 关联发布者用户并记录其身份标签（房东直租/房屋中介）。

#### Scenario: 多维度筛选查询
- **WHEN** 管理员或小程序用户按租赁类型/状态/小区/户型组合筛选
- **THEN** 系统返回符合条件的房源列表，支持分页

#### Scenario: 房源上下架管理
- **WHEN** 房源发布者或管理员执行上架/下架操作
- **THEN** 系统更新 `status` 字段（0下架/1上架），下架房源不在小程序列表展示

#### Scenario: 房源信息维护
- **WHEN** 管理员在后台编辑房源（标题、价格、户型、面积、楼层、装修、图片、描述）
- **THEN** 房源信息更新，小程序端实时展示最新数据

#### Scenario: 房东/中介标签展示
- **WHEN** 房源在小程序端展示时
- **THEN** 系统根据发布者 `user_type` 显示"房东直租"或"房屋中介"标签

### Requirement: 电话簿管理（含申请收录审批）
系统 SHALL 提供周边商家电话簿管理，支持用户申请新增商家电话。申请需登记营业执照、店主姓名、联系电话等信息，经管理员审批通过后才能在小程序展示。

#### Scenario: 用户申请收录商家
- **WHEN** 小程序用户提交商家收录申请（商家名称、店主姓名、联系电话、营业执照图片、商家分类）
- **THEN** 系统创建申请记录，状态"待审批"
- **AND WHEN** 管理员审批通过
- **THEN** 商家信息写入 `biz_phonebook` 表，状态"已审批"，小程序端可展示

#### Scenario: 管理员驳回商家申请
- **WHEN** 管理员驳回并填写原因
- **THEN** 申请状态置为"已驳回"，通知申请人

#### Scenario: 电话簿分类查询
- **WHEN** 小程序用户按商家分类（餐饮/便利店/家政/维修等）查询
- **THEN** 系统返回对应分类的商家电话列表

### Requirement: 电子合同签署与 PDF 下载
系统 SHALL 提供在线电子签名签署合同功能。租赁房源可发起合同签署流程，双方完成电子签名后系统生成 PDF 文件供下载。

#### Scenario: 发起合同签署
- **WHEN** 房东（或中介）与租客就某房源达成租赁意向后发起合同
- **THEN** 系统创建合同记录（关联房源、双方用户、租金、租期、起止时间），状态"待签署"
- **AND WHEN** 双方在合同上绘制电子签名并提交
- **THEN** 系统保存签名图片，合同状态置为"已签署"，记录签署时间

#### Scenario: 合同 PDF 下载
- **WHEN** 合同签署完成后用户请求下载 PDF
- **THEN** 系统使用模板引擎渲染合同内容（含双方签名图片），生成 PDF 文件返回给用户

## MODIFIED Requirements

### Requirement: 若依菜单权限扩展
基于若依现有菜单权限体系，新增业务菜单：小程序用户管理、房屋租赁（含小区管理、小区申请审批）、房屋管理、电话簿管理（含商家申请审批）、电子合同管理。通过 SQL 脚本插入 `sys_menu` 记录，并使用 `@PreAuthorize("@ss.hasPermi('biz:xxx:xxx')")` 注解控制接口权限。

## REMOVED Requirements
无移除项。
