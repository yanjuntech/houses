# 房屋租赁系统接口功能总结

## 1. 项目概述

房屋租赁系统是一个基于Spring Boot的后端服务，提供了完整的房屋租赁业务管理功能。系统使用MyBatis-Plus作为ORM框架，集成了Knife4j增强API文档功能，方便开发和测试。

## 2. 已实现的接口

### 2.1 用户管理接口（UserController）

| 接口路径 | 方法 | 功能描述 | 参数说明 |
|---------|------|---------|----------|
| `/user/list` | GET | 获取用户列表 | `userType`（可选）：用户类型 |
| `/user/get/{id}` | GET | 根据ID获取用户 | `id`：用户ID |
| `/user/add` | POST | 添加用户 | `User`对象 |
| `/user/update` | PUT | 更新用户 | `User`对象 |
| `/user/delete/{id}` | DELETE | 删除用户 | `id`：用户ID |
| `/user/audit` | PUT | 审核用户 | `userId`：用户ID，`status`：状态 |
| `/user/blacklist` | PUT | 管理黑名单 | `userId`：用户ID，`isBlacklisted`：是否黑名单 |

### 2.2 房屋管理接口（HouseController）

| 接口路径 | 方法 | 功能描述 | 参数说明 |
|---------|------|---------|----------|
| `/house/list` | GET | 获取房屋列表 | `houseType`（可选）：房屋类型，`status`（可选）：状态，`communityId`（可选）：小区ID，`ownerId`（可选）：房东ID |

| `/house/add` | POST | 添加房屋 | `House`对象 |
| `/house/update` | PUT | 更新房屋 | `House`对象 |
| `/house/delete/{id}` | DELETE | 删除房屋 | `id`：房屋ID |
| `/house/publish` | PUT | 上下架房屋 | `houseId`：房屋ID，`isPublished`：是否上架 |
| `/house/status` | PUT | 更新房屋状态 | `houseId`：房屋ID，`status`：状态 |

## 3. 技术实现

1. **框架**：Spring Boot 3.2.4
2. **ORM**：MyBatis-Plus 3.5.3.1
3. **安全**：Shiro 1.9.1 + Spring Security
4. **认证**：JWT
5. **数据库**：SQL Server
6. **API文档**：Knife4j 4.3.0
7. **工具类**：Hutool 5.8.25

## 4. API文档访问

项目集成了Knife4j增强API文档功能，启动项目后，可以通过以下地址访问API文档：

```
http://localhost:8080/api/doc.html
```

## 5. 总结

目前，房屋租赁系统已经实现了完整的核心功能，包括用户管理、小区管理、房屋管理、租赁管理、维修管理和标签管理等。系统使用了现代的技术栈，提供了清晰的API接口，方便前端调用。

通过Knife4j的集成，开发者可以更加方便地查看和测试API接口，提高开发效率。系统的接口设计遵循了RESTful风格，使用了统一的响应格式，便于前端处理。

未来，可以根据实际需求，进一步扩展系统功能，如添加支付功能、消息通知功能等。
